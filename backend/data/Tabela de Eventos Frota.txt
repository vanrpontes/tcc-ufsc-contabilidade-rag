TITULO: Tabela de Eventos Frota
URL: https://futura.atlassian.net/wiki/spaces/SPT/pages/2878899835
--------------------------------------------------
Nesta tabela são cadastrados os Eventos a serem utilizados no 
Controle de Despesas com Frota
.
A tela que permite manipular esta tabela é acionada a partir do Menu principal com as opções - 
Tabelas Auxiliares / Transportes / Frotas / Eventos
:
Para acrescentar um novo Evento acione o botão de inclusão de registro 
.
O Sistema exibe a tela para preenchimento dos novos dados.
O evento é o que caracteriza a despesa no momento da sua inclusão. Desta forma, boa parte das informações podem ser pré-configuradas no evento, para que, na escolha do evento quando na 
inclusão de eventos relacionados ao Frota
, elas já sejam sugeridas.
Sequencial:
 gerado automaticamente pelo sistema.
Descrição do Evento:
 denominação do evento.
Pede Local:
 se este evento solicitará um 
Local de Evento
.
Pede Quantidade:
 se este evento solicitará a quantidade (litros,kg,etc) do produto abastecido, neste caso o evento também solicitará o 
produto
 que deverá estar previamente cadastrado.
Pede Valor:
 se deverá ser informado um valor para o evento.
Movimenta Estoque:
 se deverá movimentar ou não o estoque.
Pede KM:
 se este evento pedirá a quilometragem do veículo quando da ocorrência do evento.
Pede Aplicação:
 se será solicitada qual a 
Aplicação
 da despesa.
Pede Grupo:
 se o evento solicitará o 
Grupo de Despesas
.
Pede Subgrupo:
 se o evento solicitará o 
SubGrupo de Despesas
.
Pede Funcionário:
 se o evento trará o 
Funcionário Padrão
 para este evento.
Ativo:
 permite ativar ou inativar um evento.
De acordo com o que foi informado nas opções acima, é possível já cadastrar o padrão para este evento, ou seja, o que será sugerido para o usuário na inclusão do evento. Pode ser cadastrado como padrão a 
Aplicação
 (se estiver marcado Pede Aplicação), o 
SubGrupo
 (se estiver marcado Pede Subgrupo), o 
Grupo
 (se estiver marcado Pede Grupo) e o 
Funcionário
 (se estiver marcado Pede Funcionário).
À partir da versão 8.17, caso estejam marcadas as opções 
Pede Grupo
 e 
Pede Subgrupo
, só poderá ser selecionado um Subgrupo que esteja vinculado ao respectivo Grupo, no cadastro. Para informações sobre cadastro de Subgrupo e vínculo com Grupo, acesse o link 
É possível também informar qual o 
padrão contábil
 será sugerido para este evento, ou seja, qual(is) a(s) conta(s) a ser(em) movimentada(s) versus a Conta de Adiantamento no Fechamento da Viagem. Este padrão contábil só será utilizado nas despesas cadastradas pelo 
Fechamento da Viagem
.
Acione o botão 
OK
 para atualizar os dados no Sistema.
O Sistema retorna a tela de navegação.
Para retornar ao Menu principal acione o botão 
Sair
 .
