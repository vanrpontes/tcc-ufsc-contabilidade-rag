Um ponto sobre a configuração das URLs para a gerenciadora AT&M. A diferenciação entre os ambientes é feita via token ID específico, fornecido pela própria gerenciadora.

Como utilizar
Após realizar a emissão do 
CT-e ou MDF-e,
 o SisPetro irá automaticamente enviar as informações para API da AT&M. É possível visualizar como está o processo de envio em 
Manutenção de Notas Fiscais
 / Aba Transportes / Sub Aba Averbação ou na Manutenção de MDF-e / Aba Diversos / Sub aba Averbação.
Em 
Tabelas Auxiliares / Transporte / Gerenciadora de Averbação 
é possível visualizar as comunicações e, assim como na tela acima, ver as notas enviadas com sucesso. 
Reenvio ou Cancelamento de envio de informação
Se houver algum erro de comunicação é possível fazer o reenvio da informação. Além disso, é possível também cancelar a comunicação com a AT&M. Em 
Manutenção de Notas Fiscais
 / Aba Transportes / Sub Aba Averbação há dois botões que permitem o reenvio, ou o cancelamento de envio da informação. Os botões são habilitados no perfil de usuário, 
Aba Fretes
.
Inativar a comunicação
Caso seja necessário inativar a Gerenciadora de Averbação AT&M o usuário deverá verificar primeiro se não tem 
nenhum documento
 (CT-e ou MDF-e) com Status de Envio Pendente e se a Averbadora não está vinculada na tela de Configuração de Sistema. Com essas condições sanadas a inativação será feita. É bom ressaltar que os documentos gerados até o momento da inativação ficarão na fila para ser enviados, exceto se o usuário cancele o envio.
Histórico de Comunicação
O SisPetro, por padrão, armazenará os últimos 90 dias do histórico das comunicações entre a AT&M e o SisPetro. Caso queira alterar isso, basta acessar a 
Limpeza de dados
 e modificar a quantidade de dias a preservar.
