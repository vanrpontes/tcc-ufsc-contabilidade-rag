Objetivo
Mostrar como o SisPetro utiliza a Consulta da ANP
O SisPetro utiliza uma API para realizar consultas automáticas aos postos regulamentados. Essa consulta é feita cliente a cliente e ela (quando o posto é localizado) traz informações como Autorização, Bandeira, Situação, Data da Publicação entre outras.    
Caso a consulta 
não encontre
 o CNPJ consultado isso quer dizer que o posto (cliente) em questão não está autorizado na ANP.
Como alternativa
 ao sistema de consulta automatizada, a ANP disponibiliza também uma planilha (formato 
.csv
) com todos os postos autorizados (baixada manualmente pelo usuário) para atualizar os postos no SisPetro.
A consulta via PLANILHA não traz as informações dos produtos permitidos para venda.

Validações do SisPetro após a consulta ANP
É de conhecimento que a ANP realiza a validação que a Distribuidora só pode vender para postos de bandeira Branca ou cuja bandeira seja a mesma que sua, por este motivo foi implementado o 
controle por bandeira
 e por 
atividade econômica
. 
Como fazer para o SisPetro bloquear venda para outras bandeiras?
É preciso configurar no Perfil de sistemas (
Bloqueio de pedido por Bandeira
), aba 
Vendas,
 que a empresa trabalha com o bloqueio de vendas para outras bandeiras.
Na 
Tabela Bandeiras
 é preciso preencher o campo 
Bandeira ANP, 
exatamente como está na consulta ANP. Feito isso não será possível incluir pedidos de venda para outras bandeiras. 
Para mais informações, ver 
Tabela Bandeiras
.
Como configurar o SisPetro para atualizar a Bandeira do cliente, durante a Consulta ANP?
É preciso configurar o perfil de sistemas (
Atualiza Bandeira do Cliente
). 
Para mais informações, ver 
Clientes
.
Também é preciso ter o campo 
Bandeira ANP
 na 
Tabela Bandeiras
, 
preenchidas. Caso não esteja não será atualizada.
Como configurar o SisPetro para bloquear o cliente quando a bandeira retornada da Consulta ANP não estiver cadastrada na Tabela de Bandeiras?
Como já foi mencionado, a Distribuidora só pode vender para clientes que tenham bandeira Branca ou igual à sua. Para ter certeza que não terá algum cliente regular que esteja com bandeira errada, este perfil foi criado. Caso as bandeiras estejam cadastradas corretamente, o cliente só será bloqueado caso a ANP retorne algo diferente do Cadastro de Bandeira. 
É preciso configurar o perfil de sistema da 
aba Clientes
.

Não são todos os tipos de clientes que precisam passar pela verificação da ANP. Para definir se o cliente será validado a

precisa estar habilitada para a consulta.
