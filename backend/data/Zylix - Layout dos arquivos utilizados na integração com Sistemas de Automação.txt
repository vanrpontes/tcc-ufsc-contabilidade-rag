Objetivo
Mostrar qual a informação que cada campo do arquivo de integração com a Zylix aguarda. 
note
A Zylix não possui manual para base de desenvolvimento. Foi baseado em outra integração.

Nome dos arquivos gerados pelo SisPetro
Os arquivos gerados pelo SisPetro terão sempre os mesmos nomes, exemplos, PE_00001234.txt  e PS_00001234.txt, onde:
PE = Portaria de Entrada
PS = Portaria de Saída
00001234 = É o número da 
FILA
 com zeros a esquerda 
Estrutura dos arquivos gerados pelo SisPetro
Portaria de Entrada:
text
wide
1800
Grava_Ordem              					-> Comando 
CaminhoBancoDados        C:\Users\futura 	-> caminho do banco conforme configurado no Depósito
Ordem                    134 		 		-> Sequencial da fila, sem zeros a esquerda
TipoOrdem                Descarga			-> Tipo da OC, Carregamento ou Descarga
Cliente                  Empresa			-> Nome usual da empresa no SisPetro		
Placa                    XXX-0000			-> Placa principal do caminhão
Motorista                Motorista			-> Nome do motorista
IdProduto_1              GASA				-> Código fixo do produto no Sistema de Automação (cadastrado no SisPetro no campo Código de Automação). No caso de produto composto, será o código e volume do produto composto.
Cap_Compartimento_1      30000				-> Volume alocado ao compartimento (pode ser menor que a capacidade)
Lacre_1       								-> em desuso, mas deve ser informado no arquivo, sem valor           
Portaria de Saída:
text
wide
1800
Ler_Dados                					-> Comando
CaminhoBancoDados        C:\Users\futura	-> caminho do banco conforme configurado no Depósito
Ordem                    134				-> Sequencial da fila, sem zeros a esquerda
Nome dos arquivos gerados pela automação Zylix
Os arquivos gerados pela automação Zylix seguirá o mesmo nome de arquivo gerado pelo SisPetro, com a inicial 
"R", 
que significa 
Retorno,
 exemplos, RPE_00001234.txt  e RPS_00001234.txt, onde:
RPE = Portaria de Entrada
RPS = Portaria de Saída
00001234 = É o número da 
FILA
 com zeros a esquerda
