Geral




Lentidão na Aba de Preços no Cadastro de Clientes


A aba de Preços no Cadastro de Clientes lista os preços dos produtos para o cliente selecionado e lista também o custo desses mesmos produtos. Como o custo é calculado para cada produto na medida que é solicitado pelo usuário, em alguns clientes esse processo tem se tornado lento pois é analisada a vigência de cada produto para cálculo do custo. A partir dessa versão, só iremos exibir os preços de venda,
 
caso o usuário necessite das informações de custos nessa aba, será necessário habilitar o parâmetro "Visualiza Informações de Custos" e com isso as informações ficarão sempre visíveis.

SisPetro no Windows 7

Em alguns casos onde os computadores que utilizam o SisPetro estejam utilizando o Windows versão 7, ocorre um problema de exibição dos botões que ficam na parte inferior da tela de Ordem de Carregamento. Diminuímos um pouco a tela para que os botões fiquem um pouco mais acima. Dessa forma procuramos minimizar o problema de exibição da tela no Windows 7 sem afetar os clientes que utilizam outras versões do Windows, onde o problema não ocorre.

Manutenção de Lotes - Problemas no Grid

Ao navegar pelo grid da tela de Manutenção de Lotes, ao posicionar-se no último registro, se fosse pressionado "seta para baixo" no teclado, o sistema incluía um registro vazio e, após, emitia uma mensagem de erro apagando o registro. Esse problema foi corrigido.

ALP (CCF 7376).


Fórmula do Histórico na Tela de Codigo de Contabilização

Na versão 7.1 introduzimos no SisPetro o padrão de históricos com 250 caracteres, o que possibilita o usuário colocar muito mais informações nesses campos e para uma boa visualização, esses campos são exibidos em forma de pop-up. Porém, na tela de Código de Contabilização, apesar do campo ser "Histórico" esse campo não deveria ter sido alterado para essa nova forma de visualização, permanecendo o formato antigo que abria a tela de seleção de campos para o código de contabilização. Essa alteração foi desfeita, voltando ao seu funcionamento normal. 
UPA (CCF 9758)


Apuração do Livro Fiscal

Ao incluir uma apuração do livro fiscal, o SisPetro verifica se já existe uma apuração no período informado. O problema está quando duas pessoas estão incluindo ao mesmo tempo a apuração, como no momento em que o sistema checou se já havia uma apuração, e nenhuma das duas apurações havia sido gravada, para o sistema elas não existem o que permite dois usuários estarem no mesmo processo e conseguirem depois gravar as apurações. A partir dessa versão estaremos validando também na gravação da apuração, impedindo que o usuário consiga incluir duas apurações para o mesmo período.

Movimentação de Estoque em Unidade de Medida diferente do Faturamento


Disponibilizadas configurações no Cadastro de Produtos e na Natureza de Operação para permitir a movimentação de estoque numa unidade de medida diferente da unidade de medida padrão (geralmente litro), nos casos onde a compra do produto é feita numa unidade de medida diferente da unidade padrão. Além disso foi permitido que o produto tenha várias unidades de medidas, com fatores de conversão diferenciados.
 
Recurso útil em compra de produtos numa unidade e comercialização em outra ou em compra de combustíveis em KG.
 
Help:
 
Várias unidades de compra para um mesmo produto
 
GSN (CCF 10215,10216)


OC com Vários Produtos x Balança


Quando foi implementada a Balança entendemos que se a OC tivesse produtos diferentes não teríamos como fazer a integração com a balança, já que cada produto tem uma densidade diferente. Porém, após utilização, o que foi identificado é que neste caso (de OC com mais produtos), até se pesa o caminhão (na portaria de entrada e saída) mas o que não se faz é usar o peso para chegar ao volume do produto. Por conta disso, estamos liberando a integração da balança, independente da OC ter vários produtos para obtenção do peso inicial e final do caminhão, porem não faremos a conversão do peso em volume nesses casos.
 
Help:
 
Roteiro de Entrada para Carregamento na Base

Para integração com a balança consulte antes o modelo atendido e é necessário utilizar o arquivo BalancaToledo.dll.

Não permitir Campos nulos na Baixa

No momento de realizar uma baixa de um título, algumas informações poderiam ser deixadas sem preenchimento. Com isso, o sistema gerava erros. Corrigimos esse problema para que os campos necessários sejam sempre preenchidos. 
Se houver um campo que não possa ser deixado "em branco" o SisPetro não vai finalizar a baixa e exibirá uma mensagem informando qual o nome do campo a ser preenchido.
 
ALP (CCF 7772)


Erro na Inclusão de Centro de Custos Duplicado - Contas a Pagar Receber

Foi corrigido um problema que ocorria quando eram lançados títulos de Contas a Pagar Receber com Centros de Custo Desvinculados. Quando o usuário escolhia dois ou mais Centros de Custos iguais, o sistema gerava uma mensagem de erro em inglês. Isso devido a ser um erro no banco de dados que não havia sido tratado pelo SisPetro. 
O mesmo problema ocorria com Centro de Custos Vinculados, em que a Conta e o Centro de Custos estivessem duplicados.
