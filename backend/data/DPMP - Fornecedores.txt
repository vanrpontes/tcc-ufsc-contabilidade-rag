Através do Menu 
Arquivos / Fornecedores / painel Detalhes / painel Observações
 é possível se configurar os dados necessários no Fornecedor para uso no DPMP: 
 
A regra para o DPMP é que caso o fornecedor tenha um código de Instalação registrado na ANP, este deve ser informado no campo 
Código Instalação, 
no caso o Código de Instalação 2 do DPMP. Caso não tenha, ele é considerado como um Terceiro, então deve ser informado o CNPJ/CPF no campo Identificação. Além disso, deve-se informar o 
CNAE
 (Código Nacional de Atividade Econômica) e a 
Localidade
.
Com base na tabela de Instalações da ANP, atualizamos, pelo CNPJ cadastrado para o fornecedor, o 
código de instalação
, caso ele tenha registrado. Para este campo não é possível fazer a pesquisa numa tabela, já que o número de instalações é muito grande e varia muito. Solicite ao seu fornecedor o código de instalação dele, caso ele possua ou consulte na tabela de Instalações da ANP.
O 
CNPJ/CPF
 já tínhamos cadastrado no SisPetro, como informação primordial.
O 
CNAE
 também deverá ser informado pelo fornecedor ou consultado na tabela de atividades da ANP. Não é possível pesquisar pelo SisPetro, já que o número de atividades na tabela é grande, mas entre os fornecedores deve variar entre uns dois códigos.
A 
localidade
 indica o código ANP para onde está localizado o fornecedor e deve ser primeiramente cadastrado na 
. Já fizemos um trabalho de atualização, pelo código DNC, do código ANP. Assim, a localidade está de acordo com a configuração do município do fornecedor. Se não for informada a localidade, ele assumirá que é o mesmo município do seu cadastro principal. Caso seja necessário acrescentar mais uma localidade com um código DNC já existente, é preciso incluir uma cidade como "Região" e utilizar esta região neste campo Localidade do cadastro do fornecedor.
Preenchimento Obrigatório
Alguns usuários tem esquecido de preencher essas informações ao realizar um cadastro, e isso tem gerado trabalho extra durante a geração do DPMP, pois será necessário localizar todos Clientes sem os dados preenchidos e completar o cadastro. Para evitar isso, pode-se configurar um perfil de sistema no menu 
Ferramentas / Sistema / painel Estoque
 perfil 
Força dados de Clientes/Fornecedores para uso do DPMP
Ao marcar esse perfil, os preenchimentos dessa informação se tornam obrigatórios durante o cadastramento de novos fornecedores, não sendo possível gravar antes que preencha os dados de DPMP na aba de Observações.
Bloqueando duplicado de código de Instalação
Para a geração do DPMP o Fornecedor deve estar com o campo de código de instalação preenchido com um código valido, como cada Fornecedor tem seu próprio código de instalação, temos a opção de avisar ou até mesmo bloquear o cadastro/alteração de um Fornecedor, caso já exista outro Fornecedor utilizando esse código de instalação. Para isso, pode-se configurar um perfil de sistema no menu 
Ferramentas / Sistema / 
painel Estoque

Código de Instalação ANP de Clientes e Fornecedores.
