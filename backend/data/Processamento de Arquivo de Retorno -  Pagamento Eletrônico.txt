TITULO: Processamento de Arquivo de Retorno -  Pagamento Eletrônico
URL: https://futura.atlassian.net/wiki/spaces/SPT/pages/2878899314
--------------------------------------------------
Após a 
Leitura
 deve ser feito o processamento do arquivo clicando no botão 
 
Tela de processamento do arquivo de retorno:
Informe os dados para a baixa:
Data Movimento: 
data em que o banco liberou o dinheiro do pagamento referente.
Contas Contábeis: 
Nesse campo é necessário informar as Contas Contábeis onde serão Debitados/Creditados os valores de descontos e acréscimos, no campo CC Despesas Cobrança é a conta onde será creditado o valor caso exista alguma despesa com cobrança (Exemplo: cobrança de boleto).
Documentos
 e 
históricos: 
Caso o histórico de baixa esteja configurado para aceitar fórmulas, é possível alterar a sugestão e testá-la para prever o resultado do histórico a ser salvo.
Na baixa, caso o número do documento esteja vazio, o sistema preenche o campo com o sequencial de retorno.
Baixa em Conta Transitória: 
Essa campo será preenchido apenas quando utilizado uma conta transitória para contabilização.
Pode Contabilizar: 
só estará visível para as empresas que utilizam o módulo contábil em seguida deve-se informar o código de contabilização. É possível configurar no perfil da 
empresa para que o 
código de contabilização
 seja sugerido na tela, facilitando e evitando erros de digitação no momento da baixa. E o mesmo pode ser alterado, caso for necessário.
Para configurar a sugestão do código de contabilização da tela de Baixa, deve-se ir no 
Menu 
Ferramentas/ Sistema/ painel Contab.Real.
 
Os títulos não sofrendo qualquer alteração, o processamento continuará normalmente.
Ao final é possível emitir o recibo de baixa, mas este também estará disponível para impressões futuras na tela de Manutenção de Arquivos do Pagamento Eletrônico.
