A partir da versão 8.19 o adiantamento pode trabalhar de duas formas, em relação ao Pedido de movimentação de material.
Os exemplos são baseados em empresas que trabalhem com Saldo Excedente e controlem seus PMMs pela Quantidade Paga.
Quando o PMM é controlado pela quantidade paga, só é liberado a quantidade para utilização na Ordem de Carregamento quando é pago, ou seja, quando é feito o Adiantamento Quantificado. Sendo assim veja a diferença entre os perfis:
Temos no PMM a coluna de "Qtde Alocada", ou seja, quantidade alocada que representa a quantidade disponível ou pendente para o usuário emitir uma Ordem de Carregamento. Como essa quantidade é liberada através do adiantamento quantificado, ao se pagar a Usina ou qualquer outro fornecedor, é liberado o crédito para a utilização. Imaginem o seguinte cenário:
PMM , tem saldo pendente de 5.000 L., para completar o caminhão foi inserido o Saldo Excedente de 10.000L. A Nota Fiscal foi faturada com 14.573 L. Isso significa que o Adiantamento Quantificado deverá ser na quantidade de 9.573L. Para se pagar o Saldo Excedente foi incluído o Adiantamento para os 9.573L, porém é nesta hora que pode haver confusão. Pois a quantidade alocada será de 9.573 Litros. Assim algum usuário precisa "Desalocar" tal quantidade do PMM.
Pode haver alguns usuários que se sintam confortáveis em trabalhar desta forma, porém há outros que não. Por isso foi criado a opção de não acrescentar a Quantidade Pendente no PMM. Deve ser usada no final do contrato quando se tem o Saldo Excedente. No cenário acima, se fosse escolhida tal opção o usuário não precisaria desalocar o PMM, pois não haveria saldo disponível.

    3. Adiantamentos vinculados a PMM
Também poderá ser incluído o adiantamento vinculando-o a um 
Pedido de Movimentação de Material
 (PMM), simplificando ainda o preenchimento do controle de adiantamento quantificado, pois o mesmo será preenchido automaticamente de acordo com os dados do PMM, devendo apenas informar a quantidade.
Para vincular o adiantamento ao PMM, o sistema deve estar configurado para controlar o PMM. Para isto, deve-se marcar no menu 
Ferramentas / Sistema / Compras
 
a opção 
Controla Pedido de Movimentação de Material 
e na tela de inclusão de Adiantamento marcar a opção
 "Informa PMM?"
Quando é marcado a opção 
"Informa PMM?" 
Será apresentado a tela de pesquisa onde aparecerá os PMMs que poderão ser vinculados a este adiantamento
É possível emitir um adiantamento para a quantidade integral ou parcial do pedido, possibilitando a inclusão de diversos adiantamento para o mesmo PMM.
Adiantamento x Nota Fiscal
 A nota fiscal da usina terá que ter o valor unitário compatível com o valor do adiantamento. mesmo que o PMM seja diferente.
Não poderão ser feitas transferências para os adiantamentos que estejam vinculados a PMM.
Nesse caso de adiantamento quantificado de fornecedores, todos os depósitos estarão disponíveis para seleção, diferentemente de quando o adiantamento é de clientes, onde somente depositos que permitem venda poderão ser selecionados.
É possível informar casas decimais na quantidade dos produtos, para isso vide configuração no tópico 
.
No grid irá mostrar o código do produto, a descrição do mesmo, a quantidade original do adiantamento, preço, o saldo ainda restante desse adiantamento (caso já tenha sido utilizado uma parte) e o valor total.
Ao finalizar a inclusão do adiantamento quantificado para o fornecedor, será gerado uma entrada no estoque de acordo com os dados informados.
Na manutenção de adiantamentos a fornecedores, os adiantamentos com PMM serão visualizados na cor roxa.
