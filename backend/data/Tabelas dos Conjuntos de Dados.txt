Participação dos Campos
Como é de conhecimento de todos, estão sendo feitas alterações no layout do SPED Fiscal, quanto a criação de campos e registros.
Dependendo do período que vamos entregar a escrituração o layout pode mudar de acordo com a tabela de Código de Versões, e a cada versão do layout poderá ser incluído algum campo a um registro.
 Ex.
Dados do arquivo do período de 01/12/2009 a 31/12/2009, para o SPED este arquivo deveria ser gerado no layout 2.0
No outro mês o arquivo gerado com dados no período de 01/01/2010 e 31/01/2010, para o SPED deveria ser gerado no layout 3.0, e neste deverá ser incluído o campo COD_OBS no registro C190.
Para resolvermos esta situação podemos desmarcar o campo para a exportação no arquivo seguindo os passos abaixo.
No menu Ferramentas > Tabelas do Conjunto De Dados > Escolher o Conjunto de Dados desejado, no nosso exemplo o 1 - Livro Fiscal Eletrônico - EFD
Selecionar o registro que teve alteração, clicar no botão configurações 
No campo devemos buscar o item que teve alteração no layout.
 
 
Agora é marcar ou desmarcar o perfil "Participa do Arquivo de Exportação?"

Processamento da geração do SPED
A partir da versão 10.26 a geração do SPED acontecerá através do Agenda de Tarefas e executado pela funcionalidade 
Jobs
.
No Quadro de Funções

Jobs

Estrutura do SisPetro e Banco de Dados
Quando a estrutura do SisPetro (executáveis) estão em uma máquina e o banco de dados em outra máquina é obrigatório:
Servidor Hermes e DLLs dos SPEDs devem estar na mesma máquina onde se encontra o banco de dados
O caminho da DLL obrigatóriamente deve ser a pasta onde se encontram o Servidor Hermes e as DLLs
