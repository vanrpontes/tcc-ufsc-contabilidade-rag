Nessa fase do PIX usaremos o QR Code Estático que pode ser gerado pela própria empresa, conforme determinação do Banco Central. Futuramente podemos trabalhar com o QR Code Dinâmico, que exige comunicação com o banco para a geração do mesmo
Para recebimentos a prazo / parcelados 
o envio do QR Code será pelo Faturamento (Contas a Receber), 
o QR Code é gerado após o Faturamento, um para cada parcela do Contas a Receber, sendo enviado no layout de Cobrança (seja somente com o QR Code, seja no Boleto). Ao ser recebido o pagamento, o SisPetro baixará o título do Contas a Receber correspondente. Se esse título for de Cobrança Eletrônica, gera um Comando de Baixa para o Banco.
